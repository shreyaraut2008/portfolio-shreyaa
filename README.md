## 🎯 About This Portfolio

### Why It Exists

Most developer portfolios feel like a checklist — *name, skills, projects, contact*. Done. Forgotten.

I wanted mine to feel like a **conversation**. A window into how I think, what I value, and what kind of developer I am. I built this portfolio because I believe that *how you present your work says as much about you as the work itself*.

### The Vision

> *To create a portfolio that doesn't just show what I've built — but makes visitors feel the energy behind why I build.*

Every design decision, every animation, every interaction was made with intention. The cinematic hero. The interactive skill galaxy. The AI assistant that actually answers your questions about me. These aren't gimmicks — they're a philosophy: **if I'm going to build for users, my own portfolio should be the highest expression of what I believe great UX looks like.**

## 💎 Key Features

Every feature here was built with a *why* — not just a *what*.

---

### 🎬 Cinematic Hero Section

**What it does:** Opens with a full-screen video backdrop, dramatic typography, and an animated scroll prompt. A mute/unmute control gives visitors agency over their sensory experience.

**Why it exists:** First impressions are everything. A recruiter or collaborator spending 3 seconds on a portfolio decides in those 3 seconds whether to stay. A cinematic hero earns those seconds.

**UX Benefit:** Creates immediate emotional impact. Sets the tone that this isn't an ordinary developer — and this isn't an ordinary portfolio.

---

### 🌌 Interactive Tech Galaxy (Skills Section)

**What it does:** Renders skills as an orbiting galaxy of technologies. Hovering over each skill reveals its name and category — Machine Learning, Backend, Frontend, Database, and more.

**Why it exists:** A bulleted list of technologies tells you *what* I know. An interactive galaxy shows you *how I think about* what I know — as an ecosystem, not a checkbox.

**UX Benefit:** Memorable, tactile, and rewarding to explore. Visitors actually spend time here — which means they actually learn about my skills.

---

### 🤖 Built-in AI Portfolio Assistant

**What it does:** A conversational AI widget (powered by intelligent design) lives directly on the page. Visitors can ask it: *"Who is Shreya?", "What projects has she built?", "What makes her unique?"* — and get real, contextual answers.

**Why it exists:** Because the best way to know me is to ask questions — and I can't be on every recruiter's call at once.

**UX Benefit:** Makes the portfolio interactive beyond just visuals. Recruiters and collaborators get answers *fast*, without digging through pages of content.

---

### 🎞️ Smooth Animations & Transitions

**What it does:** Every section entrance, card reveal, and hover state is animated with Framer Motion — physics-based, smooth, and never jarring.

**Why it exists:** Animations aren't decoration. Done right, they guide attention, communicate hierarchy, and make interfaces feel alive.

**UX Benefit:** Visitors stay longer because the experience feels fluid and delightful rather than static and flat.

---

### 📱 Fully Responsive Design

**What it does:** The entire portfolio adapts gracefully from a 320px mobile screen to a 4K widescreen monitor — no broken layouts, no awkward overflows.

**Why it exists:** Because great design should look great everywhere. Period.

**UX Benefit:** Whether a recruiter opens it on their phone or a developer reviews it on a ultrawide display, the experience is consistent and polished.

---

### 🏗️ "What I'm Building Now" Section

**What it does:** Showcases three active projects in progress — HackPulse (AI Hackathon Copilot), AI Dev Workspace, and AquaSentinel (AI Flood Prediction). Each shown with status tags.

**Why it exists:** Most portfolios show what's done. I show what I'm *doing right now* — because that's the truest signal of momentum and ambition.

**UX Benefit:** Gives visitors a real-time window into my thinking and energy. Makes the portfolio feel alive, not archived.

---

### 💌 Connect Section

**What it does:** A warm, visually striking contact section with direct links to GitHub, LinkedIn, Twitter, and email — no buried contact form, no friction.

**Why it exists:** The whole point of a portfolio is connection. This section makes it *effortless*.

**UX Benefit:** Every CTA is one click. No walls, no confusion, no lost opportunities.

---

### 🎨 Modern Developer Branding

**What it does:** Consistent dark premium aesthetic, custom avatar illustrations, typographic hierarchy, and a cohesive visual identity across all sections.

**Why it exists:** Developer branding matters. I am a product. This portfolio is how I package that product.

**UX Benefit:** Visitors remember the experience — and remember the name attached to it.

## ⚙️ Installation Guide

Getting this portfolio running locally takes less than 5 minutes. Follow along!

### Prerequisites

Make sure you have these installed on your machine:

- **Node.js** v18 or higher → [Download here](https://nodejs.org/)
- **npm** or **yarn** (comes with Node.js)
- **Git** → [Download here](https://git-scm.com/)

---

### Step 1 — Clone the Repository

```bash
git clone https://github.com/shreyaraut2008/shreya-portfolio.git
```

```bash
cd shreya-portfolio
```

---

### Step 2 — Install Dependencies

Using npm:
```bash
npm install
```

Or using yarn:
```bash
yarn install
```

> This installs all packages listed in `package.json` including Next.js, Tailwind, Framer Motion, and TypeScript.

---

### Step 3 — Run the Development Server

```bash
npm run dev
```

Open your browser and visit:

```
http://localhost:3000
```

The portfolio is now running locally with hot-reloading enabled.

---

### Step 4 — Build for Production

```bash
npm run build
```

This compiles and optimizes the project for production. Check the output for bundle sizes and any warnings.

---

### Step 5 — Deploy to Vercel

The easiest way to deploy is via the [Vercel CLI](https://vercel.com/docs/cli) or connecting your GitHub repo to Vercel directly.

**Option A — GitHub Integration (Recommended):**

1. Push your code to GitHub
2. Go to [vercel.com](https://vercel.com) → Import Project
3. Select your repository
4. Click **Deploy** — Vercel handles the rest automatically

**Option B — Vercel CLI:**

```bash
npm install -g vercel
vercel login
vercel --prod
```

> Every push to `main` automatically triggers a new production deployment.

## ⚡ Performance & Optimization

### Responsive Design
Built mobile-first with Tailwind's responsive breakpoint system. Every layout adapts across `sm`, `md`, `lg`, and `xl` viewports — tested from 320px to 2560px.

### SEO Considerations
Next.js's `metadata` API powers semantic `<title>`, `<meta description>`, Open Graph, and Twitter Card tags — ensuring the portfolio is discoverable and shareable across search engines and social platforms.

### Fast Loading
- Static assets (images, video) served from Vercel's global CDN edge
- Next.js automatic code-splitting ensures only the JS needed for each page is loaded
- Images use Next.js `<Image>` component with lazy loading and size optimization

### Clean Code Architecture
- TypeScript enforces type safety and self-documenting interfaces
- Components are single-responsibility — each does one thing well
- Utility-first Tailwind classes keep styling co-located with markup

### Reusable Components
Every section is an isolated React component. Need to update the Skills section? Touch one file. Want to add a new project card? One component, one prop change.

## 🎨 Design Philosophy

### Dark Premium Aesthetic
The palette — deep blacks, subtle purples, electric accents — communicates sophistication without trying too hard. Dark themes are standard in developer tools because they reduce eye strain and look incredible on modern displays.

### Cinematic Storytelling
Design choices were borrowed from film: the hero feels like an opening scene, sections transition like acts, and the final contact section is the emotional resolution. Portfolios don't have to be boring pamphlets.

### Modern Developer Branding
Custom avatar illustrations, consistent iconography, and a unified typographic system create a visual identity that's distinctly *Shreya* — not off-the-shelf, not generic.

### Smooth Interactions
Framer Motion animations are physics-based — they feel like they have weight and momentum. Nothing snaps or jumps. Every interaction earns its place by communicating something meaningful about hierarchy or state.

### User-First Experience
Every design decision was evaluated through one lens: *does this serve the visitor?* If an animation is distracting rather than guiding, it gets removed. If a section is confusing, it gets rewritten. The visitor's experience always wins.

## 🤝 Contributing

This is a personal portfolio, but the code is open for learning, inspiration, and collaboration.

### Ways to Contribute

- 🐛 **Found a bug?** Open an issue with steps to reproduce
- 💡 **Have a feature idea?** Open an issue with the `enhancement` label
- 🎨 **Want to suggest a design improvement?** Start a discussion
- ⭐ **Love the project?** Drop a star — it means more than you know

### How to Contribute

```bash
# 1. Fork the repository on GitHub

# 2. Clone your fork
git clone https://github.com/YOUR-USERNAME/shreya-portfolio.git

# 3. Create a feature branch
git checkout -b feature/your-amazing-feature

# 4. Make your changes, then commit
git commit -m "feat: add your amazing feature"

# 5. Push to your fork
git push origin feature/your-amazing-feature

# 6. Open a Pull Request on GitHub
```

### Commit Convention

This project follows [Conventional Commits](https://www.conventionalcommits.org/):

```
feat:     New feature
fix:      Bug fix
docs:     Documentation change
style:    Formatting, no logic change
refactor: Code restructure, no behavior change
perf:     Performance improvement
chore:    Build process or tooling change
```
